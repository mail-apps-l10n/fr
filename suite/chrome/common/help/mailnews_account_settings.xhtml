Paramètres des comptes courrier & groupes

Cette section décrit le paramétrage de la boîte de dialogue &accountManagerTitle.label;. Contrairement à la boîte de dialogue &prefWindow.title;, qui applique les paramètres choisis à tous les comptes, la boîte de dialogue Paramètres des comptes courrier & groupes vous permet de définir les réglages d'un compte précis.

Si la boîte de dialogue &accountManagerTitle.label; n'est pas affichée, suivez ces étapes :

  1. Ouvrez la fenêtre Courrier
  2. Ouvrez le menu &editMenu.label; et choisissez &accountManagerCmd.label;.
  3. Sélectionnez le compte dont vous désirez voir ou changer les paramètres.

Paramètres des comptes courrier & groupes - &accountTitle.label;

  1. Ouvrez le menu &editMenu.label; et sélectionnez &accountManagerCmd.label;. La boîte de dialogue &accountManagerTitle.label; s'affiche.
  2. Sélectionnez le nom du compte pour afficher les paramètres du compte.
&accountName.label;
Le nom de ce compte. Vous pouvez lui donnez le nom que vous voulez. C'est ce nom qui s'affichera dans la liste des comptes de la fenêtre principale de &brandShortName; Courrier & Groupes.
Pour tous les types de compte, sauf les comptes de flux
&identityTitle.label;
Contient votre nom, votre adresse électronique, votre adresse de réponse (uniquement si elle est différente de votre adresse électronique) et le nom de votre organisation (optionnel).

Si vous désirez apposer votre signature sur tous les messages sortants, saisissez votre dans signature cette zone de texte. Cochez la case Utiliser HTML pour pouvoir utiliser des balises HTML, par exemple, <b>bold</b> (optionnel).
Apposer la signature à partir d'un fichier
Vous permet d'apposer votre signature à partir d'un fichier (texte, HTML, ou image) plutôt que de saisir son contenu. L'utilisation de cette fonction désactive le texte saisi dans la zone de texte de la signature. Cliquez sur &choose.label; pour localiser votre fichier de signature (optionnel).

Pour les comptes de flux

Empties the Trash folder whenever you quit Mail & Newsgroups.

Paramètres des comptes courrier & groupes - Paramètres serveur

&brandShortName; Courrier & Groupes peut travailler avec deux types différents de serveurs de messagerie : IMAP et POP. Si vous n'êtes pas sûr du type de serveur que votre fournisseur d'accès à Internet supporte, posez-lui la question. Si votre fournisseur d'accès à Internet supporte les deux, les descriptions suivantes vous aideront à choisir l'un ou l'autre.

Dans cette section :

  • À propos du Internet Message Access Protocol (IMAP)
  • À propos du Post Office Protocol (POP)
  • Paramétrage d'un serveur IMAP
  • Paramétrage avancé d'un serveur IMAP
  • Paramétrage d'un serveur POP
  • Paramétrage d'un serveur de groupes

[ Retour au début de la section ]

À propos du Internet Message Access Protocol (IMAP)

Avantages : vos messages et tout changement les concernant restent sur le serveur, préservant l'espace disque de votre ordinateur. Ce serveur, qui donne accès à une boîte aux lettres toujours à jour, permet de relever vos messages depuis des endroits différents (travail, domicile, etc). La durée de la connexion est plus courte car on ne télécharge initialement que les en-têtes des messages.

Inconvénient : IMAP n'est pas supporté par tous les FAI.

[ Retour au début de la section ]

À propos du Post Office Protocol (POP)

Avantages : tous vos messages sont téléchargés entièrement sur votre ordinateur d'un seul coup. Vous pouvez cependant préciser au serveur de conserver une copie ou d'effacer les messages uniquement lorsqu'ils le sont aussi localement. POP est actuellement supporté par la plupart des FAI.

Inconvénients : si vous utilisez plusieurs ordinateurs, des messages peuvent se trouver sur l'un ou sur l'autre, mais pas sur les deux. POP ne fonctionne pas aussi bien que IMAP avec une connexion à bas débit. Vous ne pourrez pas non plus accéder à tous les dossiers de messagerie à partir de divers endroits.

Veuillez noter que les serveurs POP plus récents ont des fonctionnalités permettant de relever seulement les en-têtes plutôt que les messages complets, comme le permet IMAP. Utiliser cette fonctionnalité permet à POP d'être presque aussi rapide qu'IMAP.

Paramétrage avancé d'un serveur IMAP

Les paramètres avancés d'un serveur IMAP sont bien souvent automatiquement réglés par le serveur. Si vous n'êtes pas sûr des réglages de cette boîte de dialogue, contactez votre FAI ou votre administrateur système.

Si la boîte de dialogue des paramètres avancés IMAP n'est pas affichée, ouvrez une fenêtre Courrier .

  1. Ouvrez le menu &editMenu.label; et sélectionnez &accountManagerCmd.label;. La boîte de dialogue &accountManagerTitle.label; s'affiche.
  2. Sélectionnez le nom du compte et cliquez sur la catégorie &serverSettings.label;.
  3. Si vous avez sélectionné un serveur IMAP, vous pouvez cliquer sur &advancedButton.label; pour paramétrer les réglages additionnels, tels que :
    • Le dossier sur le serveur IMAP.
    • L'affichage des seuls dossiers avec abonnement.
    • Le support des sous-dossiers par le serveur.
    • Tous les espaces privés et publics (dossiers partagés) pour ce répertoire.

Pour plus d'informations, voir Ajouter et retirer des annuaires LDAP.

&useHtml.label;
Vous pouvez utiliser le format HTML par défaut pour écrire vos messages (courriers ou groupes de discussion). Laissez la case décochée pour utiliser le format texte par défaut. Les messages en HTML peuvent contenir du texte formaté (couleur, police, etc.), des liens, des images ou encore des tableaux, tout comme une page Web. Cependant, avant d'utiliser ce format, vérifier que les destinataires seront capables de lire les messages en HTML avec leur client de courrier électronique. De nombreux groupes de discussions ou listes de diffusion n'acceptent que les messages au format texte.

Astuce : si vous désirez utiliser l'un des formats de façon occasionnelle, vous pouvez maintenir la touche Majuscule enfoncée lorsque vous cliquerez sur l'un des boutons Écrire, &replyMsgCmd.label; ou &replyToAllMsgCmd.label; pour basculer automatiquement dans le format secondaire qui ne sera utilisé que pour le présent message.

Vous pouvez choisir de le placer avant ou après votre réponse, sachant qu'il est plus naturel et fortement recommandé de le laisser avant. Vous pouvez également choisir de sélectionner automatiquement le texte cité.

Disk Space Settings (Blogs)

Messages from blogs & news feeds accounts are only stored in your local machine in their short form, ie. the article summary, regardless of whether your settings are to show the full articles by default. Still, there are options to control how much disk space is used by your blogs & news feeds account. If you are not already viewing the Disk Space preferences for a blogs & news feeds account, follow these steps:

Begin from the Mail window.

  1. Open the Edit menu, choose Mail & Newsgroups Account Settings. You see the Mail & Newsgroups Account Settings dialog box.
  2. Click the Disk Space category for a blogs & news feeds account.

There, you can specify which messages should be deleted to recover disk space:

  • Don't delete any messages: Select this option to keep all messages forever. Keep in mind that, if you are subscribed to very high-traffic blogs, this will increase the occupied disk space steadily and could eventually fill up your hard disk.
  • Delete all but the most recent [____] messages: Select this option to keep in each feed only a maximum number of messages. Enter the maximum number of messages (being 1,000 by default).
  • Delete messages more than [__] days old: Select this option to keep in each feed only messages that are not older than the number of days you enter here (being 30 days by default).
  • Always keep flagged messages: Check this option to save (not delete) flagged messages, regardless of its age.

Common Retention Policy Settings

&brandShortName; peut automatiquement effacer des anciens messages pour vous. Vous pouvez configurer ce processus à l'aide des options listées sous &retentionCleanup.label; :

&retentionKeepAll.label;
Garde tous les messages et n'efface jamais un message automatiquement.
&retentionKeepRecent.label; [__] &message.label;
Entrez le nombre de messages à garder. Avec ce paramètre, seuls les messages plus âgés que ces messages seront effacés.
&retentionKeepMsg.label; [__] &daysOld.label; jours :
Garde tous les messages qui sont arrivés dans la période spécifiée.

À l'aide de deux derniers paramètres, vous pouvez limiter encore plus les trois options pour effacer automatiquement des messages. Ceci est spécialement utile en combinaison avec l'option de garder tous les messages.

&retentionApplyToFlagged.label;
Use this option to deny &brandShortName; to delete any messages you have flagged.
&nntpRemoveMsgBody.label; [__] &daysOld.label;
Sélectionnez cette option pour empêcher &brandShortName; d'effacer les messages qui sont plus récents que la période spécifiée.

This policy can be overridden for an individual folder in the Folder Properties, Retention Policy tab.

Note: If message synchronization is enabled (for IMAP), or messages are left on the server for POP accounts), the settings apply to both local copies and their originals on the server.

Paramètres des comptes courrier & groupes - &securityTitle.label;

Cette section décrit comment configurer les paramètres d'un compte courrier ou groupe qui contrôlent la sécurité des messages. Avant que vous ne le fassiez, il vous faudra obtenir un ou plusieurs certificats de courrier électronique. Pour les détails, voir Signature & Chiffrement des messages.

Si les paramètres de sécurité d'un compte courrier ne sont pas affichés, ouvrez une fenêtre Courrier :

  1. Ouvrez le menu &editMenu.label; et choisissez &accountManagerCmd.label;.
  2. Cliquez sur &securityTitle.label; en-dessous du nom d'un compte de messagerie dont vous désirez configurer les paramètres de sécurité.

Dans cette section

  • À propos des certificats
  • Signature numérique
  • Chiffrement

À propos des certificats

Le panneau &securityTab.label; est principalement utilisé pour sélectionner deux certificats :

  • Le certificat personnel que vous voulez utiliser pour signer les messages que vous envoyez.
  • Le certificat que vous désirez que vos correspondants utilisent pour chiffrer les messages qu'ils vous envoient.

L'utilisation d'un ou de deux certificats dépend de la politique de votre autorité de certification (AC) qui valide votre certificat. Vous pouvez utiliser un seul certificat pour les deux objets ou deux certificats différents. Même si vous n'en avez qu'un, vous devez le spécifiez deux fois, une fois pour la signature numérique et une fois pour le chiffrement.

Les certificats sélectionnés ici seront inclus dans tous les messages signés que vous enverrez. Ces certificats permettront à vos destinataires de vérifier votre signature numérique et de chiffrer les messages qu'ils vous enverront.

Signature numérique

Vous pouvez utiliser le panneau Sécurité pour spécifier la manière de signer vos messages :

&signMessage.label;
Cochez cette case si vous souhaitez signer numériquement tous les messages que vous envoyez (un certificat personnel doit être spécifié avant de pouvoir cocher cette case.).
&signingCert.message;
Si le champ est vide ou s'il affiche un mauvais certificat, cliquez sur Sélectionner pour en choisir un autre.

En ne tenant pas compte de la case à cocher &signMessage.label;, vous pouvez changer d'avis avant d'envoyer un message.

Pour changer le paramètre de signature numérique pour un message que vous écrivez dans la fenêtre de rédaction, cliquez sur la petite flèche sous l'icône &securityTab.label; dans la barre d'outils et sélectionnez ou désélectionnez &menu_securitySign.label;. Pour plus de détails, voir Signer numériquement et chiffrer un nouveau message.

Chiffrement

Vous pouvez utiliser le panneau &securityTab.label; pour spécifier comment utiliser habituellement le chiffrement lors de l'envoi de messages :

&neverEncrypt.label;
Sélectionnez cette option si vous ne voulez jamais utiliser le chiffrement, ou seulement occasionnellement.
&alwaysEncryptMessage.label;
C'est l'option à sélectionner lorsque l'on souhaite utiliser systématiquement le chiffrement. Si vous n'avez pas tous les certificats nécessaires, le message ne sera pas envoyé à moins que vous ne désactiviez le chiffrement pour ce message en particulier.
&encryptionCert.message;
Si le champ est vide ou s'il affiche un mauvais certificat, cliquez sur &encryption.certificate.button; pour en choisir un autre.

En ne tenant pas compte de l'option de chiffrement choisie, vous pouvez changer d'avis avant d'envoyer un message.

Pour changer le paramètre de signature numérique pour un message que vous écrivez dans la fenêtre de rédaction, cliquez sur la petite flèche sous l'icône &securityTab.label; dans la barre d'outils et sélectionnez ou désélectionnez &menu_securitySign.label;. Pour plus de détails, voir Signer numériquement et chiffrer un nouveau message.

Paramètres des comptes courrier & groupes - Serveur sortant (SMTP)

Le serveur sortant envoie vos messages à leurs destinataires. Même si vous avez plusieurs comptes de messagerie ou de groupe, vous n'avez généralement besoin de spécifier qu'un seul serveur sortant (SMTP) pour vous permettre d'envoyer vos messages.

Si les paramètres du serveur sortant ne sont pas affichés, ouvrez une fenêtre Courrier & Groupes :

  1. Ouvrez le menu &editMenu.label; et choisissez &accountManagerCmd.label;. La boîte de dialogue &accountManagerTitle.label; s'affiche.
  2. Sélectionnez la catégorie Serveur sortant (SMTP). Vous pouvez alors ajouter un nouveau serveur sortant ou modifier les paramètres d'un serveur existant suivant les indications ci-dessous. Si vous n'êtes pas sûr des paramètres à utiliser, contactez votre FAI, ou votre administrateur système.
&serverDescription.label;
Une courte description libre à propos de la configuration du serveur. Ceci s'affichera dans la liste des serveurs.
&serverName.label;
&brandShortName; Courrier & Groupes utilise le nom de serveur sortant (SMTP) que vous avez entré en répondant aux questions de l'assistant de création d'un compte. Pour changer le nom du serveur sortant (SMTP), saisissez-le dans le champ &serverName.label;.
&serverPort.label;
Si nécessaire, entrez le numéro de port par lequel se connectera le serveur SMTP. Si rien n'est spécifié, le port par défaut est utilisé. Changez-le si le serveur de courrier sortant attend les connexions sur un port non standard.
&userName.label;
Si votre serveur SMTP requiert une authentification pour envoyer un message, sélectionnez cette option afin que &brandShortName; Courrier & Groupes vous demande d'entrer votre nom d'utilisateur et votre mot de passe d'authentification, puis entrez votre nom d'utilisateur. La première fois que vous enverrez un message, vous serez invité à entrer votre mot de passe d'authentification, vous pourrez alors l'enregistrer pour les futures sessions.
&authMethod.label;
Il existe deux méthodes pour établir un connexion sécurisée à un serveur sortant. Choisissez celle qui est supportée par votre serveur (si vous choisissez une méthode non supportée par votre serveur, vous recevrez un message d'erreur lors de l'envoi de courrier).

Si votre serveur sortant SMTP est configuré pour envoyer des messages chiffrés, choisissez parmi les options suivantes (en cas de mauvaise configuration, un message d'erreur s'affichera chaque fois que vous enverrez un message) :

&connectionSecurity.label; :
Sélectionnez le type de connexion à utiliser (sécurisée ou non) dans la liste déroulante.
